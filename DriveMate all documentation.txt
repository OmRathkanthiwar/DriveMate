# Driver Rental Platform - Complete MERN Stack Development Guide

# Project Overview

A MERN stack platform where customers can hire professional drivers to drive their own vehicles.

The system supports:

- Customer ride booking
- Driver registration and verification
- Admin management system
- OTP-based ride start/end
- Online & cash payments
- Real-time ride updates
- Driver matching using location radius
- Fare calculation using day/night rates

# Step 2 - Fare Calculation

Backend calculates:

- Distance
- Total Hours
- Day Hours
- Night Hours

---

# Example Pricing

```txt
Day Rate = ₹100/hour
Night Rate = ₹150/hour
Night Allowance = ₹200
Minimum Fare = ₹500
```

---

# Fare Formula

```txt
Fare =
(dayHours × dayRate)
+
(nightHours × nightRate)
+
nightAllowance
```

---

# Example Calculation

## Ride Details

```txt
Total Hours = 8
Day Hours = 5
Night Hours = 3
```

---

## Calculation

```txt
(5 × 100) + (3 × 150) + 200

= 500 + 450 + 200

= ₹1150
```

---

# Minimum Fare Logic

```txt
if (fare < minimumFare)
    fare = minimumFare
```

---

# Night Time Logic

```txt
Night Timing:
10 PM → 6 AM
```

Hours inside this range:
- use night rate
- apply night allowance

---

# Step 3 - Driver Matching

## Conditions

Driver should be:

- Verified
- Active
- Available
- Inside customer radius
- Matching vehicle preference

---

# Driver Radius Logic

Driver sets:

```txt
Maximum Radius = 15 KM
```

Only rides within this radius are shown to the driver.

---

# Step 4 - Ride Acceptance

## Flow

1. Customer books ride
2. Ride request created
3. Nearby drivers notified
4. Driver accepts ride
5. Driver becomes BUSY
6. Other drivers cannot accept same ride

---

# Step 5 - OTP Verification

# Ride Start OTP

## Flow

1. Backend generates OTP
2. OTP sent via Twilio SMS
3. Customer tells OTP to driver
4. Driver enters OTP
5. Ride starts

---

# Ride End OTP

## Flow

1. Backend generates new OTP
2. Customer shares OTP
3. Driver enters OTP
4. Ride ends

---

# OTP Rules

```txt
OTP expiry = 5 minutes
```

---

# Step 6 - Payment System

# Payment Types

```txt
1. Razorpay
2. Cash
```

---

# Razorpay Logic

```txt
20% → Admin
80% → Driver
```

---

# Cash Payment Logic

- Driver collects payment
- Driver must pay admin commission within 2 days
- If not paid:
  
```txt
driver.status = blocked
```
